## 概要

Altair Framework の Automation 機能（Blockly ビジュアルプログラミング）では、以下の操作を行う際に自動的に運動学計算を実行して、4 個の MDD モータの目標回転速度を算出します：

- ゲームパッド（PS5 コントローラー）の入力
- スライダー・ジョイスティックの操作
- Python コードでの `robot_kinematics()` ブロック実行

運動学計算により、**任意の前後・左右・旋回速度指令から、各モータの理想的な回転速度（rps）を逆算**して、目的の移動軌跡を実現します。

## 運動学モデル

### 座標系定義

```
      +Y (前方/北)
       ↑
       |  Ω (旋回角速度)
       |  ⤴ (反時計回り=正)
    ←─+─→ +X (右方/東)
       |
```

### 基本パラメータ

| パラメータ | 記号 | 単位 | 説明 |
|:---|:---:|:---:|---|
| **前後速度** | $V_y$ | mm/s | 正=前進、負=後退 |
| **左右速度** | $V_x$ | mm/s | 正=右移動、負=左移動 |
| **旋回角速度** | $\Omega$ | rad/s | 正=反時計回り（左旋回） |
| **旋回直径** | $L$ | mm | ロボット中心回りの旋回半径を決める外部パラメータ |
| **車輪径** | $d$ | mm | 各モータのホイール直径（モータごとに個別設定可能） |

### 共通計算式

#### 1. 旋回項の係数

$$r_t = \frac{L}{2000}$$

$L$ は GUI で指定する「旋回直径（mm）」です。**$L / 2000$ で mm → m に変換**され、旋回時のモータ速度差を決めます。

#### 2. 回転速度への換算係数

各モータについて、**移動速度（mm/s）から回転速度（rps）** への換算係数：

$$k_{wheel} = \frac{1000}{\pi \times d}$$

ここで：
- $d$ = 該当モータの車輪径（mm）
- 結果の単位は rps（revolutions per second = 回/秒）

**例：** 車輪径 100mm の場合
$$k = \frac{1000}{\pi \times 100} \approx 3.183 \text{ (rps/(mm/s))}$$

## 4つのロボット構成タイプ

### タイプ 1: 3輪オムニ (3_OMNI)

**ホイール配置：** 120°間隔で3つのオムニホイール

```
          M1 (前)
           △
          /|\
         / | \
        /  |  \
       /   |   \
    M2○────┼────○M3
      120° | 120°
           |
```

#### 運動学式

$$v_1 = V_y + r_t \cdot \Omega$$

$$v_2 = -0.866025 \times V_x - 0.5 \times V_y + r_t \cdot \Omega$$

$$v_3 = 0.866025 \times V_x - 0.5 \times V_y + r_t \cdot \Omega$$

#### 逆運動学計算の流れ

```
入力: Vx, Vy, Ω
  ↓
各輪の理想速度 v1, v2, v3 を上記式で計算
  ↓
車輪径から rps に換算: rps = v * k_wheel
  ↓
MDD.set_targets([rps1, rps2, rps3, 0])
```

#### 特徴

- **最小回転半径が小さい**（120°オムニの等方性）
- 全方向移動可能
- 簡潔な構成

---

### タイプ 2: 4輪オムニ (4_OMNI)

**ホイール配置：** 45°間隔で4つのオムニホイール（正方形配置）

```
    LF ◇─────────◇ RF
      ╱           ╲ 45°
     ╱             ╲
    │      +Y       │
    │       ↑       │
 ───┼───────+───────┼──→ +X
    │               │
     ╲             ╱
      ╲           ╱
    LR ◇─────────◇ RR
```

#### 運動学式

$$v_{LF} = (-0.707107 \times V_x + 0.707107 \times V_y - r_t \cdot \Omega) \times k_{LF}$$

$$v_{RF} = (0.707107 \times V_x + 0.707107 \times V_y + r_t \cdot \Omega) \times k_{RF}$$

$$v_{LR} = (-0.707107 \times V_x - 0.707107 \times V_y + r_t \cdot \Omega) \times k_{LR}$$

$$v_{RR} = (0.707107 \times V_x - 0.707107 \times V_y - r_t \cdot \Omega) \times k_{RR}$$

ここで $0.707107 = \frac{1}{\sqrt{2}} = \cos(45°)$

#### 特徴

- 全方向移動・横スライド・その場旋回が可能
- 4輪構成で安定性が高い
- 45°オムニホイールの標準的な構成

#### 動作例

| 操作 | $V_x$ | $V_y$ | $\Omega$ | 結果 |
|:---|:---:|:---:|:---:|---|
| 前進 | 0 | 正 | 0 | 4輪同速回転 → 前進 |
| 右移動 | 正 | 0 | 0 | 斜め（LF, RR 逆、LR, RF 逆）|
| 左旋回 | 0 | 0 | 正 | LF, LR は減速、RF, RR は加速 |
| 斜め前右 | 正 | 正 | 0 | 複合移動 |

---

### タイプ 3: 4輪メカナム (4_MECANUM)

**ホイール配置：** ローラー付きメカナムホイール（正方形配置）

```
    LF □─────────□ RF
      │           │
      │    MDD    │
      │    M1-M4  │
      │           │
    LR □─────────□ RR
```

#### 運動学式

$$v_{LF} = (V_y - V_x - r_t \cdot \Omega) \times k_{LF}$$

$$v_{RF} = (V_y + V_x + r_t \cdot \Omega) \times k_{RF}$$

$$v_{LR} = (V_y + V_x - r_t \cdot \Omega) \times k_{LR}$$

$$v_{RR} = (V_y - V_x + r_t \cdot \Omega) \times k_{RR}$$

#### 特徴

- **最も一般的な全方向移動ロボット構成**
- メカナムホイールのローラーにより、全方向速度を実現
- 制御式がシンプル（オムニより計算量が少ない）
- 実装が容易

#### 動作例

| 操作 | $V_x$ | $V_y$ | $\Omega$ | 結果 |
|:---|:---:|:---:|:---:|---|
| 前進 | 0 | 正 | 0 | 4輪同速 |
| 右移動 | 正 | 0 | 0 | LF, RR は逆、LR, RF は逆 |
| 左旋回 | 0 | 0 | 正 | LF, LR は逆、RF, RR は同速 |

---

### タイプ 4: 差動2輪 (2_DIFF)

**ホイール配置：** 左右 2 個のホイールグループ（タンク型）

```
    L ◯─────────◯ R
      │           │
      │   中心    │
      │           │
    L ◯─────────◯ R
```

#### 運動学式

$$v_L = V_y - r_t \cdot \Omega$$

$$v_R = V_y + r_t \cdot \Omega$$

左右各グループのモータは同じ速度を受け取ります：
- LF, LR → $v_L$ 
- RF, RR → $v_R$

#### 特徴

- **最もシンプルな制御方式**
- 前後移動と旋回のみ（横スライド不可）
- 信頼性が高い、制御が容易
- リソース消費が少ない

#### 動作例

| 操作 | $V_x$ | $V_y$ | $\Omega$ | 結果 |
|:---|:---:|:---:|:---:|---|
| 前進 | 0 | 正 | 0 | $v_L = v_R$ → 直進 |
| 左旋回 | 0 | 0 | 正 | $v_L < v_R$ → 左旋回 |
| 前進+左旋 | 0 | 正 | 正 | $v_L$ 低速、$v_R$ 高速 → 前左カーブ |

## GUI パラメータ設定

### Dashboard タブ（各モータの個別設定）

#### 1. 制御モード選択

```
Mode: [Speed ▼] [Angle ▼] [Pos ▼]
```

- **Speed（速度モード）：** 目標値を rps（回転数/秒）として指定
  - 値の範囲: -10 ～ +10 rps
  - 運動学ブロックはこのモードで動作

- **Angle（角度モード）：** 目標値を度（deg）として指定
  - 値の範囲: -360 ～ +360°

- **Pos（位置モード）：** 目標値をミリメートル（mm）として指定
  - 値の範囲: -1000 ～ +1000 mm

#### 2. 車輪径 (Diameter)

```
Diameter: [100.0] mm
```

- **用途：** mm/s → rps への換算係数を決定
- **計算式：** $k = \frac{1000}{\pi \times d}$
- **推奨値：** ロボットの実際のホイール径を測定して入力
- **デフォルト：** 100 mm

**例：** タイヤが 60mm ならば `60` と入力

#### 3. 回転方向（Direction）

```
Direction: [+1 ▼] [-1 ▼]
```

- **+1：** 正の目標値で正回転（通常）
- **-1：** 正の目標値で逆回転（反転）

**用途：** ロボット設計上、左右のホイール回転方向の不整合を補正

#### 4. PID ゲイン（モータ速度制御用）

```
P: [10.0]  I: [0.0]  D: [0.0]
```

- **P（比例）：** 現在値と目標値の差に比例した補正
- **I（積分）：** 目標値との誤差を積分した補正
- **D（微分）：** 誤差の変化速度に基づく補正

**リアルタイム調整：** ± ボタンで増減可能。モータが振動したら P を下げるなど、実走行中に調整可能

### Automation タブ（運動学制御ブロック設定）

#### 1. ロボット構成タイプ選択

```
🤖 ロボット運動学制御
Type: [4輪メカナム ▼]
Options:
  - 3輪オムニ
  - 4輪オムニ
  - 4輪メカナム
  - 差動2輪
```

#### 2. モータドライバ選択

```
モータドライバ: [drive_mdd ▼]
```

対象となる MDD モジュール名を選択

#### 3. 旋回直径 (L)

```
旋回直径: [300] mm
```

- **用途：** 旋回時の角速度 → 各モータ速度差への換算
- **計算式：** $r_t = \frac{L}{2000}$
- **物理的意味：** ロボット中心を通る回転軸との距離
- **推奨値：** ロボットの軸間距離（左右ホイール間の距離）
  - 例：100mm 幅のロボット → 100mm
  - 例：200mm 間隔 → 200mm

**目安：** 大きい値 → 旋回がゆっくり、小さい値 → 旋回が速くなる

#### 4. モータ割当（LF / RF / LR / RR）

```
LF/左前/左輪: [M1 ▼]
RF/右前/右輪: [M2 ▼]
LR/左後:      [M3 ▼]
RR/右後:      [M4 ▼]
```

- **LF = Left Front（左前）**
- **RF = Right Front（右前）**
- **LR = Left Rear（左後）**
- **RR = Right Rear（右後）**

各ホイール位置に対応する MDD 内のモータ（M1～M4）を指定

**例：** メカナムロボットで M1=左前、M2=右前、M3=左後、M4=右後の場合：
```
LF: M1
RF: M2
LR: M3
RR: M4
```

#### 5. モータ反転チェック（INV）

```
LF 反転: [ ] (チェック = 反転)
RF 反転: [ ]
LR 反転: [ ]
RR 反転: [ ]
```

- **チェック入れる：** 該当モータの回転方向を反転
- 用途：ロボット内部のモータ取付方向の違いを補正

#### 6. 入力軸の設定

```
入力 Vx (左右速度 mm/s): [gamepad_axis_scaled: AXIS 0 × 1000]
入力 Vy (前後速度 mm/s): [gamepad_axis_scaled: AXIS 1 × 1000]
入力 ω (旋回角速度 rad/s): [gamepad_axis_scaled: AXIS 2 × 0.01]
```

ゲームパッドの各軸を選択し、スケーリング係数を設定

## 座標系と入出力仕様

### 入力座標系（ゲームパッド・スライダー）

| 軸 | ゲームパッド | デフォルト | 単位 | 意味 |
|:---|:---:|:---:|:---:|---|
| **Vx** | Axis 0 (R-Stick ←→) | ×1000 | mm/s | 正 = 右移動、負 = 左移動 |
| **Vy** | Axis 1 (R-Stick ↑↓) | ×1000 | mm/s | 正 = 前進、負 = 後退 |
| **ω** | Axis 2 (L-Stick ←→) | ×0.01 | rad/s | 正 = 反時計回り（左旋回） |

**例：** PS5 コントローラーでの操作
- 右スティック上 → Vy = 1000 mm/s (最大前進)
- 右スティック下 → Vy = -1000 mm/s (最大後退)
- 左スティック左 → ω = -0.1 rad/s (左旋回)

### 出力（各モータ目標速度）

運動学計算後、**MddClient.set_targets([rps1, rps2, rps3, rps4])** で送信

| インデックス | 対応 | 単位 | 範囲 |
|:---:|:---:|:---:|:---:|
| 0 | M1 | rps | -10 ～ +10 |
| 1 | M2 | rps | -10 ～ +10 |
| 2 | M3 | rps | -10 ～ +10 |
| 3 | M4 | rps | -10 ～ +10 |

## 使用例

### 例1: 4輪メカナムロボット（標準的な構成）

**ロボット仕様：**
- 車輪配置：LF / RF / LR / RR (M1 / M2 / M3 / M4)
- ホイール径：80 mm（すべて同じ）
- 旋回直径：200 mm

**GUI 設定：**
```
Dashboard:
  M1: Mode=Speed, Diameter=80, Direction=+1
  M2: Mode=Speed, Diameter=80, Direction=+1
  M3: Mode=Speed, Diameter=80, Direction=+1
  M4: Mode=Speed, Diameter=80, Direction=+1

Automation (Blockly):
  Type: 4輪メカナム
  MDD: drive_mdd
  旋回直径: 200 mm
  LF: M1, RF: M2, LR: M3, RR: M4
  全反転なし
```

**操作例：**
```
ゲームパッド入力: Vx=500, Vy=800, ω=0
     ↓
運動学計算:
  v_LF = (800 - 500) * 3.979 = 1191.3 rps
  v_RF = (800 + 500) * 3.979 = 5170.9 rps
  v_LR = (800 + 500) * 3.979 = 5170.9 rps
  v_RR = (800 - 500) * 3.979 = 1191.3 rps
     ↓
MDD.set_targets([1.191, 5.171, 5.171, 1.191])
```

結果：**右前方への移動**

### 例2: 3輪オムニロボット

**ロボット仕様：**
- ホイール配置：M1 (前) / M2 (右奥) / M3 (左奥)
- ホイール径：100 mm
- 旋回直径：300 mm

**GUI 設定：**
```
Automation (Blockly):
  Type: 3輪オムニ
  MDD: drive_mdd
  旋回直径: 300 mm
  LF: M1, RF: M2, LR: M3, RR: none
```

**操作例（右移動）：**
```
入力: Vx=500, Vy=0, ω=0

v1 = 0 + 0 = 0 rps        (前輪)
v2 = (-0.866 * 500) * 3.183 = -1376.2 rps  (右奥)
v3 = (0.866 * 500) * 3.183 = 1376.2 rps   (左奥)

MDD.set_targets([0, -1.376, 1.376])
```

結果：**右移動**（前輪は停止、奥のホイールが反対方向）

## 注意事項

### 単位の一貫性

- **距離系** mm（ミリメートル）
- **速度系** mm/s（ミリメートル毎秒）
- **角度系** rad/s（ラジアン毎秒）
- **回転** rps（回転数毎秒）

### 車輪径の精度

実測値を正確に測定してください。誤差が大きいと、横スライドや旋回軌跡が崩れます。

### PID ゲイン調整

- 最初は**デフォルト値（P=10.0, I=0.0, D=0.0）** で試す
- モータが目標値に追従できない場合は **P を増加**
- 振動・オーバーシュートが起きる場合は **P を減少** または **D を増加**

### 反転フラグの使用

ロボット組立時点で、ホイール取付方向を統一してから、反転フラグは使わないことが推奨されます。ただし緊急時の補正には有効です。
